# 카카오톡 클론코딩

## 프로젝트 설명


### 📆기간



2023.07월~08월



### 💡카카오톡을 클론코딩 한 이유



첫 클론코딩입니다. 완성도를 높이기 위해 접근하기 쉽고 단순한 사이트를 찾았습니다.
HTML과 CSS의 다양한 기능을 활용해볼 수 있었습니다. 



### 💻 시연 영상

https://youtu.be/BryO7inuBbY



## 💬 기능 설명

### 1. 로그인 페이지

<img width="612" alt="스크린샷 2023-09-26 오후 5 44 04" src="https://github.com/KimRosa/kokoa-clone-2023/assets/108081283/b5252846-fa46-4e23-a5d0-c65322e8bd93">


- 모든 화면에 공통으로 들어가는 상단바와 하단바를 구현했습니다. 
- 하단바의 경우, 클릭하면 색상이 변하며 해당 페이지로 넘어갈 수 있습니다. 

- 로그인의 아이디와 비밀번호 폼을 선택하면 회색이 노란색으로 바뀌며 유저의 시각적 인식이 유연하도록 구현했습니다.
- 로그인 버튼이 눌리면 친구 페이지로 넘어갑니다. 


### 2. 친구 페이지

<img width="686" alt="스크린샷 2023-09-26 오후 5 57 37" src="https://github.com/KimRosa/kokoa-clone-2023/assets/108081283/597b7a1f-df45-4a27-bc68-8cde9ecf883e">

- 친구, 채팅, 찾기 , 더보기 페이지에 공통으로 들어가는 페이지 제목과 우측의 기본 아이콘 클래스를 생성합니다.
- 채팅과 찾기 페이지에 공통으로 사용되는 사용자 이미지와 이름을 표기하는 칸에 대한 클래스를 생성합니다. 


### 3. 채팅 페이지

<img width="686" alt="스크린샷 2023-09-26 오후 5 51 25" src="https://github.com/KimRosa/kokoa-clone-2023/assets/108081283/bb52dbd8-9d26-456d-abed-dda9df633641">

- 친구페이지에서 사용한 클래스를 응용합니다. 메세지를 남길 수 있는 폼을 추가했습니다.
- 새로운 채팅을 확인할 수 있는 1마크를 구현해볼 수 있었습니다.

  
### 4. 찾기 페이지

## 아쉬웠던 점

- 아무래도 HTML과 CSS만으로 화면 구현에 목적을 둔 과제였기 때문에 화면의 유동성이 없다는 점이 아쉬웠습니다. 
또한 웹페이지에 어플인 카카오톡을 구현하고자 했기 때문에 폼들 사이의 거리감에 어색함이 있었다고 생각합니다. 
홀로 도전한 첫 클론코딩이었기 때문에 만족합니다만, 다음에는 더 다양하고 풍성한 정보를 가지고 있는 화면을 구현해보고싶습니다.
- git을 제대로 활용하지 못했습니다. 클론코딩 이후 git의 사용법을 조금은 알게되었기 때문에 다음 코딩에서는 잘 응용하고싶습니다. 

## 느낀 점
- 클론코딩은 처음이라 그런지 이론으로만 접했던 HTML과 CSS를 실전으로 써볼 수 있다는 것 자체로 즐거웠습니다.  비록 100% 완벽히 구현하진 못했지만 이번 프로젝트를 계기로 더 성장할 수 있으리라 생각합니다.
